introduction

The game is a word guessing game. Players choose letters from the pool to build a word represents the picture on the screen.

This project is developing a customized HTML5 version of a popular mobile game "疯狂猜图" (translated as "Guess Pic Mania") for a vehicle serveice company.

reflection

It's already been more than two years since I developed this game. Both web technology and my understanding/skill on web development improved a lot. If I would develop this game again, there might be several changes:

  • Use responsive fuild layout instead of viewport. When I developed this game, iPhone 4 is the most popular smartphone, and screen ratio of phones is not really an issue. So I can use viewport and zoom to achieve responsive. As iPhone 5, 6 came out, screen ratio differs a lot. Using percentage size should be a better solution for responsive layout.
  • Separate view and control. I might use a MVC/MV* framework to separate view and control.
  • Modularize code. The JS code did a fairly good job on usability. However, it still exposes too many variables. And I need to read through each part to recall how it works.
  • Develop a better API to backend. When I developed this game, I was not very familiar with ajax. So it took the backend developper and me quite long time to change the code and connect to real data. If I would develop it again, I would definitely develop a better API for it.
  • Performance improvement. Though performance is not a big issue for a simple app like this. I still want to improve it. This version did a poor work on rendering performance. There are too many operations on DOM, which causes re-rendering of the page. (I had almost no idea about performance back then...)
  • Coding style. I might also change the coding style a little bit.
